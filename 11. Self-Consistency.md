### Post-Mortem Black Friday com Dados Reais

**CENÁRIO:** Sistema de e-commerce ficou completamente fora do ar por 45 minutos durante a Black Friday (14:15-15:00), causando prejuízo de R$ 500 mil. Sistema atende 15k usuários simultâneos no pico.

**CONTEXTO:** Deploy realizado 16h antes do incident com otimizações de performance e nova feature de recomendações. Tráfego subiu de 1,200 para 4,100 RPS antes da falha.

- **Hipótese 1: Infrastructure Overload**
- **Hipótese 2: Database Bottleneck** 
- **Hipótese 3: Application Memory Leak**
- **Hipótese 4: External Dependencies Failure**

Para cada hipótese, siga este raciocínio passo a passo:

1. **Evidências que suportam esta hipótese:** Quais logs, métricas e dados específicos confirmam esta teoria?
2. **Timeline de falha:** Como esta root cause explica a sequência de eventos de 14:10 até 14:18?
3. **Mecanismo de cascade failure:** Como esta causa inicial levou ao complete outage?
4. **Correlação com o deploy:** Como as mudanças realizadas 16h antes contribuíram para esta falha?
5. **Explicação das anomalias:** Como esta hipótese explica métricas anômalas (ex: CDN cache miss spike)?
6. **Fatores que contradizem:** Quais dados NÃO se alinham perfeitamente com esta teoria?

**Etapa 2 – Avaliação comparativa das hipóteses:**

Compare as quatro hipóteses com base nos critérios:

- **Consistência com dados:** Qual hipótese explica melhor todas as evidências?
- **Simplicidade da explicação:** Qual requer menos suposições?
- **Timeline alignment:** Qual se alinha melhor com a sequência temporal?
- **Correlation strength:** Qual tem correlação mais forte com o deploy recente?
- **Cascade mechanism:** Qual explica melhor o complete system failure?
- **Anomaly explanation:** Qual explica melhor as métricas inesperadas?

Classifique cada hipótese como Alta/Média/Baixa probabilidade para cada critério.

**Etapa 3 – Determinação da root cause definitiva:**

- Com base na análise das etapas anteriores, determine a root cause mais provável
- Justifique detalhadamente por que esta hipótese é a mais consistente com os dados
- Identifique fatores contribuintes secundários que amplificaram o impact
- Proponha um plano de prevenção específico baseado na root cause identificada
