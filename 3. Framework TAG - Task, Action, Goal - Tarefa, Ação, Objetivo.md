## Framework TAG — Task, Action, Goal (Tarefa, Ação, Objetivo)

O TAG é ideal quando você precisa que o modelo não só explique algo, mas elabore um plano de ação com objetivo claro e mensurável.

### Quando usar
- Planejamento de estratégias técnicas (CI/CD, migração, segurança)
- Quando o resultado precisa convencer stakeholders
- Situações onde o objetivo final é quantificável

### Quando não usar
- Quando você não tem um objetivo claro definido (defina antes de usar o framework)
- Análises exploratórias sem meta específica

---

## ❌ Forma Errada

```
Como fazer CI/CD?
```

## ✅ Forma Certa

**Task (Tarefa)**: Definir uma estratégia de CI/CD para nossa startup de fintech.

**Action (Ação)**: Elabore um plano detalhado explicando as etapas necessárias, ferramentas recomendadas, e como convencer a diretoria sobre os benefícios do investimento.

**Goal (Objetivo)**: Reduzir o tempo de deploy de 2 dias para 30 minutos e diminuir bugs em produção em 80%.
