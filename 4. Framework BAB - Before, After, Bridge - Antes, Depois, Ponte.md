## Framework BAB — Before, After, Bridge (Antes, Depois, Ponte)

O BAB é um framework narrativo. Ele descreve a situação atual (dor), o estado desejado (visão) e pede ao modelo que construa o caminho entre os dois. É especialmente eficaz para problemas de transformação e melhoria.

### Quando usar
- Problemas onde a dor atual e o estado desejado são claros
- Propostas de melhoria para apresentar a stakeholders
- Situações onde você quer que o modelo entenda o contexto emocional/operacional do problema

### Quando não usar
- Quando você não tem clareza sobre o estado desejado (defina o "depois" antes)
- Tarefas técnicas pontuais sem contexto de transformação

---

## ❌ Forma Errada

```
Melhore meu monitoramento
```

## ✅ Forma Certa

**Before (Antes)**: Nossa equipe descobre problemas de performance apenas quando clientes reclamam. Não temos visibilidade das métricas de sistema e perdemos receita por downtime não detectado.

**After (Depois)**: Quero ter visibilidade completa da saúde dos sistemas, receber alertas proativos antes que usuários sejam impactados, e ter relatórios executivos mensais sobre disponibilidade.

**Bridge (Ponte)**: Me ajude a definir uma estratégia de monitoramento que inclua: quais métricas acompanhar, como estruturar alertas eficazes, e como apresentar dados para diferentes stakeholders.
