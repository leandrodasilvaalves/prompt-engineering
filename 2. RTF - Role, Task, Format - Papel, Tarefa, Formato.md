## Framework RTF — Role, Task, Format (Papel, Tarefa, Formato)

O RTF é o framework mais fundamental de engenharia de prompt. Ele garante que o modelo saiba quem deve ser, o que fazer e como entregar o resultado.

### Quando usar
- Ponto de partida para qualquer prompt mais elaborado
- Quando você precisa de uma análise ou documento estruturado
- Consultorias, relatórios e recomendações técnicas

### Quando não usar
- Tarefas simples e diretas onde o contexto é óbvio
- Quando você já tem um framework mais específico para o caso (ex: BAB para narrativas de mudança)

---

## ❌ Forma Errada

```
Me ajude com Docker
```

## ✅ Forma Certa

**Role (Papel)**: Você é um consultor sênior em estratégia de containerização com experiência em otimização de infraestrutura.

**Task (Tarefa)**: Analise os prós e contras de migrar nossa arquitetura monolítica atual para containers Docker, considerando nossa equipe de 8 desenvolvedores e orçamento limitado.

**Format (Formato)**: Apresente uma análise estruturada com: benefícios, desafios, custos estimados, timeline de implementação e recomendações finais.
